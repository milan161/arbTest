# 滚动轮问题经验总结

## 问题场景
Dashboard 页面的表格数据量大（国内LOF TAB有很多基金），导致：
1. 水平滚动条被遮挡，只有滚动到表格最底部才能看到
2. 垂直滚动条有两层，外层和内层各有一个，交互混乱

## 根本原因
`n-data-table` 组件内部有自己的滚动机制，当设置 `scroll-x` 时会创建独立的水平滚动容器。外层 `.table-scroll-wrapper` 只设置了 `overflow-x: auto`，没有高度限制，导致：
- 垂直滚动由表格内部处理
- 水平滚动由外层处理
- 两个滚动容器分离，用户体验差

## 解决方案

### 方案一：外层容器统一处理滚动（推荐）

```css
.table-scroll-wrapper {
  width: 100%;
  height: calc(100vh - 220px); /* 固定高度，让滚动条始终可见 */
  overflow: auto; /* 同时处理水平和垂直滚动 */
}
```

**关键点：**
- 使用固定 `height` 而非 `max-height`，确保滚动条始终在同一位置
- 调整 `100vh - 220px` 中的常数来控制表格高度
- 表格内部不设置 `scroll-x`，让外层容器统一处理

### 方案二：保留表格内部滚动（备用）

如果需要保留表格的固定列功能（如 `fixed: 'right'`），可以：
1. 保留 `scroll-x` 属性
2. 给外层容器设置足够大的高度
3. 接受两层滚动的交互方式

### 方案三：同步内外层滚动（高级）

使用 JavaScript 监听滚动事件，同步内外层滚动位置：

```javascript
// 在 mounted 中添加
const wrapper = document.querySelector('.table-scroll-wrapper')
const table = document.querySelector('.n-data-table-wrapper')

if (wrapper && table) {
  wrapper.addEventListener('scroll', () => {
    table.scrollTop = wrapper.scrollTop
    table.scrollLeft = wrapper.scrollLeft
  })
  table.addEventListener('scroll', () => {
    wrapper.scrollTop = table.scrollTop
    wrapper.scrollLeft = table.scrollLeft
  })
}
```

## 注意事项

### 1. 固定列冲突
- 当表格列设置 `fixed: 'right'` 或 `fixed: 'left'` 时，必须保留 `scroll-x`
- 否则固定列会失效

### 2. 表格渲染问题
- 直接用 CSS 禁用 `n-data-table-wrapper` 的 `overflow`（设为 `visible`）会导致表格数据不显示
- 原因：`n-data-table` 依赖内部滚动容器来计算行高和渲染

### 3. 高度计算
- 使用 `calc(100vh - X)` 时，X 需要根据页面布局调整
- 太小：表格太高，滚动条跑到屏幕外
- 太大：表格太矮，显示区域不够

### 4. 交互体验
- 两层滚动容器时，鼠标位置决定哪个滚动条响应
- 鼠标在表格内：内层滚动条响应
- 鼠标在表格外：外层滚动条响应

## 最佳实践

1. **单一滚动容器原则**：尽量让一个容器处理所有滚动
2. **固定高度优先**：使用 `height` 而非 `max-height`，确保滚动条位置稳定
3. **测试多场景**：在数据量不同的 TAB 间切换测试
4. **避免 CSS 强制禁用内部滚动**：`n-data-table` 的滚动机制复杂，直接禁用会导致渲染问题
5. **Git 保存版本**：每次修改前 commit，方便回退

## 2026-07-07 列宽优化

### 问题
- `现价`/`涨跌幅`/`T-2/1日净值`/`净值日期` 没设 width，flex 撑满剩余空间，太宽浪费
- `指数价`(48px)/`指数涨跌`(54px)/`指数名称`(60px) 太窄，数据换行
- 部分列设了固定宽度后，剩余的 auto-width 列（名称/实时估值/静态估值）反而吸收全部剩余空间变得更宽

### 原则
所有列都设固定宽度，naive-ui DataTable 列不设 width 会 flex 平分剩余空间，导致列宽不可控。
