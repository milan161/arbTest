## 系统概述

### 设计目标

1. **实时性**：毫秒级行情更新，捕捉瞬间套利机会
2. **准确性**：三重估值体系交叉验证，降低单一数据源风险
3. **稳定性**：多数据源自动切换，单点故障不影响整体
4. **可扩展性**：模块化设计，易于添加新基金或数据源

### 核心创新

- **"四驾马车"配置体系**：估值、对冲、交易分离，灵活应对复杂场景
- **物理兑换比算法**：基于基准日精确计算，0网络请求极速反推
- **安全快照模式**：无行情权限时自动降级，60秒稳定更新

### 💻 核心系统架构 

#### 1. 后端路由与服务中心 (LOF02_fetch_trade_data.py)
这是整个系统的**核心生命线**。
- **通信层**：采用 Flask 提供 RESTful API 响应单次请求，结合 `flask_socketio` 提供 WebSocket 长连接，实现行情数据的“推/拉”双工模式。
- **守护线程群**：
  - `IBReader`：维护盈透 TWS/Gateway 长连接，负责盘前/夜盘美股 ETF 行情（具备无数据 60 秒兜底降级机制）。
  - `SinaFuturesReader` & `SSEFuturesReader`：维护 CME 期货和国内沪银的实时轮询与 Server-Sent Events (SSE) 流接收。
  - `LOFPriceReader`：负责 A 股 LOF 盘口数据，优先级为：QMT Socket 推送 > 通达信内存快照 > 新浪 API 兜底。

#### 2. 交易网关设计模式 (readers/trade_manager.py)
系统采用了经典的**策略模式与门面模式(Facade)**重构交易模块：
1. **统一入口**：主程序只调用 `trade_manager.send_order(broker, action, symbol, volume, price)`，不关心底层逻辑。
2. **依赖隔离**：所有的 `xtquant`、`socket` 和 `tqcenter` 导入都在该管理器内部进行。即使某个客户端未开启（例如国金 QMT 报错），也不会导致主进程崩溃，而是安全降级。

#### 3. 前端渲染分层架构 (LOF03_generate_monitor_html.py)
为了处理极度复杂的单页面应用 (SPA)，03 主文件演变成了一个“包工头”，具体工作委派给以下三大小弟：

#### 3.1 ConfigManager (LOF031)
- **职责**：解析 `lof_config.yaml`。
- **能力**：处理跨区接力估值配置、基础仓位、汇率及期货动态挂钩锚点。

#### 3.2 DataProcessor (LOF032)
- **职责**：数据清洗与 Pandas 数据框处理。
- **能力**：读取 T-1 甚至更早基准日的官方披露净值、处理节假日跳空缺口、对齐历史 ETF 与期货的基准收盘价，为前端注入高精度基准数据池。

#### 3.3 HtmlGenerator (LOF033)
- **职责**：UI 拼装与样式注入。
- **能力**：管理多达几百行的原生 CSS 与高度复杂的 JS 实时推演逻辑（Sandbox）。实现了完全不依赖后端服务器的“前端 O(1) 复杂度极速沙盘估算”。

#### 4. 健壮性与防崩溃机制
- **静默防刷屏**：底层高频行情推送与终端日志打印解耦，日志实行“冷却时间限制（如30s）”，还开发者一个整洁的控制台。
- **端口占用提示**：Flask 5000 端口如果被历史残留进程占用，系统会通过捕获 `OSError` 拦截报错，并提供中文排障指南。
- **自动降级(Fallback)**：如果 IB 失去夜盘订阅权限，自动平滑退级到 1 分钟历史 K 线快照抓取；如果 QMT 通信失败，自动降级至通达信或只读模式。

## 估值算法详解

### 静态官方估值

#### 数学推导

设：
- $N_0$ = 基准日LOF净值
- $E_{i,0}$ = 第i个ETF基准日价格
- $E_{i,t}$ = 第i个ETF T日价格
- $w_i$ = 第i个ETF权重（Σ$w_i$ = 1）
- $R_0$ = 基准日汇率
- $R_t$ = T日汇率
- $α$ = 权益仓位比例

**步骤1：计算加权价格因子**
$$F = \sum_{i} w_i \times \frac{E_{i,t}}{E_{i,0}}$$

**步骤2：计算净值变化比例**
$$\Delta = \alpha \times \left( F \times \frac{R_t}{R_0} - 1 \right)$$

**步骤3：计算静态估值**
$$N_t = N_0 \times (1 + \Delta)$$

#### 代码实现

```python
def calculate_static_nav(base_nav, base_prices, current_prices, 
                         weights, base_rate, current_rate, equity_ratio):
    """
    计算静态官方估值
    
    Args:
        base_nav: 基准日净值
        base_prices: 基准日ETF价格字典 {symbol: price}
        current_prices: 当前ETF价格字典 {symbol: price}
        weights: ETF权重列表 [(symbol, weight), ...]
        base_rate: 基准日汇率
        current_rate: 当前汇率
        equity_ratio: 权益仓位比例(0-1)
    
    Returns:
        float: 静态估值
    """
    # 计算加权价格因子
    price_factor = sum(
        weight * (current_prices[sym] / base_prices[sym])
        for sym, weight in weights
    )
    
    # 计算净值变化比例
    nav_change = equity_ratio * (price_factor * current_rate / base_rate - 1)
    
    # 计算静态估值
    static_nav = base_nav * (1 + nav_change)
    
    return static_nav
```

### 期货校准估值

#### 原理

期货价格与现货价格存在基差，通过校准值将期货价格映射为等效ETF价格：

$$E_{calibrated} = \frac{F_{future}}{C}$$

其中：
- $F_{future}$ = 期货实时价格
- $C$ = 动态校准值（由历史数据回归得出）

#### 校准值计算

```python
def calculate_calibration_value(etf_prices, future_prices, window=20):
    """
    计算期货校准值
    
    校准值 = 期货价格 / ETF价格 的历史均值
    """
    ratios = [f / e for f, e in zip(future_prices, etf_prices)]
    calibration = sum(ratios[-window:]) / window
    return calibration
```

### 纯期货原生估值

直接使用期货涨跌幅计算估值，不经过ETF转换：

$$N_{future} = N_0 \times \left[ 1 + \alpha \times \beta \times \left( \frac{F_t}{F_{settle}} - 1 \right) \times \frac{R_t}{R_0} \right]$$

其中：
- $β$ = Beta系数（原油通常0.85）
- $F_{settle}$ = 昨日结算价

## 对冲计算原理

### 核心概念：对冲值（Hedge Value）

**定义**：1股美股对冲ETF等价于多少股A股LOF基金

#### 单标的映射

$$H = \frac{E_0 \times R_0}{N_0 \times \alpha}$$

#### 多标的加权映射

$$\bar{E}_0 = \sum_{i} w_i \times E_{i,0}$$

$$H = \frac{\bar{E}_0 \times R_0}{N_0 \times \alpha}$$

### 盘中极速反推

#### 算法步骤

```
输入：计划投入本金 P，LOF实时买价 P_lof，对冲值 H

1. 理论LOF数量 = P / P_lof
2. 美股做空数量 = round(理论LOF数量 / H, 0)
3. A股买入数量 = round(美股做空数量 × H, -2)

输出：美股数量 N_us，A股数量 N_cn
```

#### 代码实现

```python
def calculate_hedge_quantity(investment, lof_buy_price, hedge_value):
    """
    计算对冲数量
    
    Args:
        investment: 计划投入本金（人民币）
        lof_buy_price: LOF实时买价
        hedge_value: 对冲值
    
    Returns:
        tuple: (美股做空数量, A股买入数量)
    """
    # 步骤1：计算理论LOF数量
    theoretical_lof = investment / lof_buy_price
    
    # 步骤2：确定美股整单（取整）
    us_shares = round(theoretical_lof / hedge_value)
    
    # 步骤3：反推A股百股（取整到百位）
    cn_shares = round(us_shares * hedge_value, -2)
    
    return us_shares, cn_shares
```

#### 优势分析

1. **0网络请求**：不依赖实时汇率
2. **无视夜盘滑点**：基于基准日精确计算
3. **物理价值匹配**：双边持仓价值误差最小化

## 扩展性设计

### 添加新基金

1. 在 `lof_config.yaml` 中添加配置：
```yaml
funds:
  - code: '新基金代码'
    name: 基金名称
    category: 类别
    trade_etf: ETF代码
    trade_future: 期货代码
    valuation_portfolio:
      - symbol: ETF1
        weight: 权重
        anchor: US
```

2. 运行 `LOF011` 和 `LOF012` 获取历史数据

3. 重启 `LOF02` 加载新配置

### 添加新数据源

1. 在 `LOF02_fetch_trade_data.py` 中实现数据获取类
2. 继承基类，实现 `fetch()` 方法
3. 在数据融合层添加优先级配置
