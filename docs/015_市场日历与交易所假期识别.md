# 市场日历与交易所假期识别

> 最后更新：2026-07-07
> AI 排查问题时，优先阅读此文档了解交易所判断逻辑，再查看代码。

---

## 1. 背景与动机

QDII 基金篮子中可能同时包含多个市场的资产：

| 基金 | 篮子成分 | 涉及交易所 |
|------|---------|-----------|
| 162411（华宝油气） | XOP | 美股 (NYSE) |
| 164701（添富贵金） | GLD + ^GLD-EU + ^GLD-JP | 美股 + 瑞士 + 日本 |
| 其他亚洲 QDII | INDA + ^INDA-HK + ^GLD-EU | 美股 + 香港 + 瑞士 |

**问题**：各交易所节假日不同。比如 2026-07-03 美国独立日，美股休市但 A 股、港股、日本、瑞士都正常开市。如果程序统一判断"今天不是交易日"就会漏掉 A 股的实时数据，或者统一按"今天开市"去拉美股数据会得到无效行情。

**解决**：对每个基金篮子组件，按它的后缀（`-EU`/`-JP`/`-HK`/裸符号）映射到对应交易所，独立判断是否开市。

| 交易所代码 | 名称 | 日历来源 | 依赖库 |
|-----------|------|---------|--------|
| `A_SHARE` | A 股（沪深） | `chinese_calendar`（含调休补班） | `chinese-calendar` |
| `NYSE` | 纽约证券交易所 | `exchange_calendars` | `exchange-calendars>=4.13.2` |

### 3.1 兜底策略

如果 `exchange_calendars` 加载失败（首次使用才 lazy-load）：

- 所有非 A 股交易所 → 仅根据周末过滤（`d.weekday() < 5`），会漏掉法定节假日
- A 股 → 先 `chinese_calendar`，再硬编码 `_HOLIDAYS_2026` 列表兜底

### 3.2 验证依赖

```python
import exchange_calendars as ec

```

2026-07-03 测试结果确认：

```
NYSE:    False（美国独立日 ✅）
XHKG:    True
JPX:     True
XSWX:    True
A_SHARE: True
```

## 4. 符号 → 交易所映射规则（`symbol_to_exchange()`）

这是整个系统的**核心函数**，决定了一个篮子组件属于哪个交易所。

### 4.1 映射优先级（从上到下依次匹配）

| 优先级 | 符号模式 | 例子 | → 交易所 | 说明 |
|--------|---------|------|---------|------|
| 1 | 以 `-EU` 结尾 | `^GLD-EU`, `^INDA-EU` | `XSWX` | 欧洲（瑞士）锚点 |
| 1 | 以 `-JP` 结尾 | `^GLD-JP`, `^INDA-JP` | `JPX` | 日本锚点 |
| 1 | 以 `-HK` 结尾 | `^INDA-HK`, `^USO-HK` | `XHKG` | 香港锚点 |
| 2 | 以 `HSI`/`HSTECH`/`HSCEI`/`HSCI`/`HSCCI`/`HSSCNE`/`HSSI`/`HSMI`/`HSSFML25` 开头 | `HSI`, `HSCEI`, `HSCI` | `XHKG` | 港股指数 |
| 3 | 以 `SZ`/`SH` 开头 | `SZ159560`, `SH000300` | `A_SHARE` | A 股代码 |
| 3 | 纯 6 位数字 | `399300`, `159560`, `930914` | `A_SHARE` | A 股/CSI 指数 |
| 3 | 以 `399`/`000`/`001`/`159`/`930`/`931`/`932` 开头 | `399006`, `930914.CSI` | `A_SHARE` | 兜底 A 股前缀 |
| 4 | 裸 ETF 符号（`GLD`/`XOP`/`SPY`/`QQQ` 等 80+ 只映射） | `GLD` → `NYSE`, `QQQ` → `NASDAQ` | 映射表 | 见 `_US_ETF_EXCHANGE_MAP` |
| 4 | 2-5 位纯大写字母 | `XOP`, `EEM`, `EWJ` | `NYSE` | 默认美股 |
| 5 | 无法识别 | 其他格式 | `None` | 日志 debug 级别记录 |

### 4.2 关键设计决策

- **后缀优先**：`^GLD-EU` 即使基础符号是 `GLD`，也按后缀映射到 `XSWX`（瑞士）
- **去 `^` 前缀**：符号清理时去除 `^`，所以 `^GLD` → `GLD` → NYSE
- **港股指数在美股兜底之前**：防止 `HSI` 被当做 3 字母美股代码匹配到 NYSE
- **裸 ETF 兜底**：任何 2-5 位纯大写字母都被当做美股（NYSE），这是一个约定

### 4.3 美股 ETF 交易所映射表

定义在 `_US_ETF_EXCHANGE_MAP` 字典中：

```python
_US_ETF_EXCHANGE_MAP = {
    'XOP': 'NYSE', 'GLD': 'NYSE', 'USO': 'NYSE', 'SLV': 'NYSE',
    'SPY': 'NYSE', 'INDA': 'NYSE', 'EEM': 'NYSE', 'EWJ': 'NYSE',
    # ... 其他 NYSE ETF
    'QQQ': 'NASDAQ', 'SOXX': 'NASDAQ', 'SMH': 'NASDAQ',
    'ARKK': 'NASDAQ', 'KWEB': 'NASDAQ', 'TQQQ': 'NASDAQ',
    # ...
}
```

如果新增 ETF 不在表中，默认走 NYSE（第 4 优先级兜底规则）。如果需要明确分到 NASDAQ，请在此字典中添加。

## 5. 核心函数速查

### 5.1 `symbol_to_exchange(symbol)` → 交易所代码

```python
from arbcore.utils.market_calendar import symbol_to_exchange

symbol_to_exchange('^GLD-EU')   # 'XSWX'
symbol_to_exchange('^GLD-JP')   # 'JPX'
symbol_to_exchange('^INDA-HK')  # 'XHKG'
symbol_to_exchange('GLD')       # 'NYSE'
symbol_to_exchange('QQQ')       # 'NASDAQ'
symbol_to_exchange('HSI')       # 'XHKG'
symbol_to_exchange('399300')    # 'A_SHARE'
symbol_to_exchange('930914.CSI')# 'A_SHARE'
symbol_to_exchange('XOP')       # 'NYSE'
symbol_to_exchange('-')         # None
```

### 5.2 `is_trading_day(exchange, date)` → bool

```python
from arbcore.utils.market_calendar import is_trading_day
from datetime import date

is_trading_day('NYSE', date(2026, 7, 3))   # False (独立日)
is_trading_day('XHKG', date(2026, 7, 3))   # True
is_trading_day('A_SHARE', date(2026, 7, 3))# True
```

### 5.3 `filter_closed_markets(symbols, date)` → (closed, open)

按交易所分组，返回闭市/开市的两组符号：

```python
from arbcore.utils.market_calendar import filter_closed_markets

symbols = ['GLD', '^GLD-EU', '^GLD-JP', '^INDA-HK', 'HSI', '399300']
closed, opened = filter_closed_markets(symbols, date(2026, 7, 3))
# closed = ['GLD']  （美股休市）
# opened = ['^GLD-EU', '^GLD-JP', '^INDA-HK', 'HSI', '399300']
```

### 5.4 `classify_by_exchange(symbols)` → dict

将符号按交易所分组（不判断是否开市）：

```python
from arbcore.utils.market_calendar import classify_by_exchange

classify_by_exchange(['GLD', '^GLD-EU', 'HSI', '399300'])
# {'NYSE': ['GLD'], 'XSWX': ['^GLD-EU'], 'XHKG': ['HSI'], 'A_SHARE': ['399300']}
```

## 6. 在 fund_service.py 中的集成

### 6.1 `prefetch_index_changes()`（约 541 行）

之前：有一个 `is_a_share_trading_day()` + 小时硬编码判断。

现在：对每个符号用 `symbol_to_exchange()` 分配到交易所，再用 `is_trading_day()` 独立判断：

```
for each exchange:
    if not is_trading_day(exchange, today):
        → 该交易所所有指数走 DB 兜底（index_history）
        continue
    if A_SHARE and hour >= 15:
        → 收盘了，走 DB 兜底
    elif XHKG and hour >= 16:
        → 收盘了，走 DB 兜底
    else:
        → 走实时 API 获取
```

主要变化：**不再一刀切判断"今天是不是交易日"**，而是每个交易所独立处理。

### 6.2 篮子 ETF 实时估值（约 1284 行）

在每个篮子组件的 ETF 价格获取循环中，新增交易所检查：

```python
for item in portfolio:
    raw_sym = item.get('symbol', '')
    sym_base = raw_sym.replace('^', '')
    # 去掉后缀得到基础代码
    ...
    ex = symbol_to_exchange(raw_sym)
    if ex and not is_trading_day(ex, now_dt.date()):
        logger.debug(f"跳过 {raw_sym}（{ex} 今日休市）")
        continue  # 跳过实时行情获取
    q = self.market_data_service.get_realtime_quote(sym_base)
    ...
```

**影响**：
- 如果美股假期，但篮子有 `^GLD-EU`（瑞士）→ 瑞士今天开市则正常获取
- 如果日本假期，`^GLD-JP` 被跳过，但仍尝试获取 GLD（美股）和 `^GLD-EU`
- 如果所有组件都休市，`current_etfs` 为空，标记 `_basket_missing_etf = True`

### 6.3 日志标识

- `[MARKET-CAL]` — market_calendar.py 自身的日志（如加载成功/失败）
- `[INDEX-DB]` — `prefetch_index_changes` 中某交易所非交易日走 DB 兜底
- 篮子组件跳过时用 `debug` 级别日志（`f"跳过 {raw_sym}（{ex} 今日休市）"`）

## 7. 常见故障排查

### 7.1 某个交易所的假期判断不正确

**原因 1**：`exchange_calendars` 库数据不是实时更新。它内置了官方 WMR 日历数据，但若某市场临时调整（如台风/暴雨/紧急休市），库数据不会实时反映。

**排查**：
```python
import exchange_calendars as ec
cal = ec.get_calendar('XHKG')
print(cal.is_session('2026-07-07'))  # 用目标日期验证
```
如果库数据确实有误，需要在 `_HOLIDAYS_2026` 或新增 `_EXCEPTIONAL_HOLIDAYS` 做硬编码覆盖。

**原因 2**：`exchange_calendars` 加载失败（日志查 `[MARKET-CAL] exchange_calendars 加载失败`），所有非 A 股交易所退化为仅周末判断。

**修复**：`pip install exchange-calendars` 确认安装正常。

### 7.2 某个符号映射到错误的交易所

**排查**：直接在代码中测试 `symbol_to_exchange('XXX')` 的输出。

常见陷阱：
- 港股指数（HSCI/HSSI/HSMI 等）必须在前缀列表 `_HK_INDEX_PREFIXES` 中，否则会被当做 4 位纯字母兜底到 NYSE
- 新增的 CSI 指数（如 932xxx）需要加入 `_A_SHARE_PREFIXES`
- 新 ETF 默认进 NYSE，如需要 NASDAQ 必须在 `_US_ETF_EXCHANGE_MAP` 中明确列出

### 7.3 A 股调休上班日与假期

`chinese_calendar` 库通常能正确处理调休补班（如国庆前的周末上班）。但如果该库过时，会漏掉补班。

**兜底**：2026 年法定假日硬编码在 `_HOLIDAYS_2026` 中。2027 年需要更新此列表。

### 7.4 基金不关心某些交易所

这是正常行为：162411（华宝油气）的篮子只有 XOP，`symbol_to_exchange('XOP')` → `NYSE`，只看美股日历，完全不碰香港/日本/瑞士的判断。

## 9. 维护注意事项

### 9.1 新增 ETF 符号

如果新增了一个美股 ETF 符号（如 `VOO`），`symbol_to_exchange('VOO')` 会走兜底规则 → 3 位纯大写字母 → `NYSE`。如果不对，在 `_US_ETF_EXCHANGE_MAP` 中添加。

### 9.2 更新年假列表

`_HOLIDAYS_2026` 只在 `chinese_calendar` 不可用时起效。每年初需要添加新一年份的硬编码假期。

### 9.3 exchange_calendars 版本更新

`exchange_calendars` 每年会发布包含新日历数据的版本。如果出现了某市场法定假期与库判断不一致，先尝试升级：
```bash
pip install --upgrade exchange-calendars
```

### 9.4 代码修改要求

根据 `AGENTS.md` 规则，任何 AI 修改本文件中的映射规则或函数逻辑时，必须在修改处添加注释 `# [AI-{日期}] {描述}`。
