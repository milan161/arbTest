# LOF012_generate_lof_data.py 功能分析

## 主要功能
- 生成LOF基金的历史数据文件
- 计算静态官方估值
- 从`GLD_USO_basic_data.csv`文件中复制校准数据到LOF历史文件

## 运行流程
1. **读取配置**：读取`lof_config.yaml`配置文件，获取基金列表和配置信息
2. **遍历基金**：对每个基金执行以下操作：
   - **数据更新检查**：基于文件中的最新日期，判断是否需要更新数据
   - **数据获取**：
     - 从新浪获取LOF基金的历史收盘价格数据
     - 从东财获取LOF基金的历史净值数据
   - **数据合并**：合并从不同来源获取的数据
   - **校准数据处理**：读取`GLD_USO_basic_data.csv`文件，获取校准数据
   - **估值计算**：调用`calculate_static_valuation`函数计算静态官方估值
   - **校准数据复制**：从`basic_df`中复制校准数据到LOF历史数据
   - **数据保存**：保存合并后的数据到`LOF_{fund_code}_history.csv`文件

## 校准数据处理逻辑
- **黄金类基金**（160719, 164701, 165513, 161116, 161226）：
  - 从`basic_df`中复制"黄金校准"列
  - **计算校准值**：使用公式 `(GLD价格*GLD权重 + ^GLD-EU价格*GLD-EU权重 + ^GLD-JP价格*GLD-JP权重) * 人民币中间价 / 净值`
  - **计算对冲值**：使用公式 `校准值 / 仓位`
- **原油类基金**（501018, 160723, 161129）：
  - 从`basic_df`中复制"原油校准"列
  - **计算校准值**：使用公式 `(USO价格*USO权重 + ^USO-EU价格*USO-EU权重 + ^USO-JP价格*USO-JP权重 + ^USO-HK价格*USO-HK权重) * 人民币中间价 / 净值`
  - **计算对冲值**：使用公式 `校准值 / 仓位`
- **特定基金**（162411, 161127, 161125, 161130）：从`basic_df`中复制对应的校准和对冲列（如"162411校准"和"162411对冲"），并将其重命名为"校准"和"对冲"

## 校准值和对冲值计算细节
1. **百分比处理**：权重和仓位都是百分比值，计算时会自动除以100进行转换
2. **列名处理**：支持两种格式的权重列名（带空格如"GLD 权重"和不带空格如"GLD权重"）
3. **数据填充**：对缺失数据使用0进行填充，确保计算的连续性
4. **列位置**：校准值和对冲值列会被放置在"期货结算价"列的左侧
5. **净值选择**：优先使用"LOF净值"列，若不存在则使用"净值"列

## 静态官方估值计算
1. **复制校准数据**：从`basic_df`中复制校准数据到`merged_df`
2. **基准日期寻找**：寻找有LOF净值和所有必要ETF收盘价数据的日期作为基准日期
3. **估值计算**：
   - 基于基准日期计算ETF静态估值、期货静态估值和指数静态估值
   - 计算变化比例、估值误差和溢价
4. **数据更新**：更新`merged_df`中的估值相关列

## 数据保存
- 截断过长历史，保留最近90个交易日的数据
- 确保日期列是字符串格式
- 保存数据到CSV文件

## 核心价值
- **数据整合**：将从不同来源获取的数据整合到一起，形成完整的LOF基金历史数据
- **校准数据复制**：确保LOF基金历史文件中包含最新的校准数据
- **静态估值计算**：提供基于基准日期的静态官方估值，帮助分析基金表现
- **数据标准化**：统一数据格式和处理逻辑，确保数据的一致性和可靠性
