# 🚀 LOF套利系统快速使用指南

除了使用 `LOF_start_lof_system.bat` 一键启动外，在开发和排错期间，强烈建议按以下步骤手动启动并验证各模块。
这套流程对初学者非常友好，能够清晰地看到数据是如何一步步流转的。

### 第一步：获取API基础数据
打开 CMD 终端并切换到项目目录，运行 011 脚本：
```cmd
cd /d D:\Study\codexTest\CodexLOFarb
python LOF011_generate_basic_data.py
```
**验证点**：去 `data\woodyAPI\` 目录下，检查是否生成了今天最新的 `.json` 和 `.csv`，正常生成即代表成功！

### 第二步：获取并对账LOF历史数据
在同一个 CMD 窗口继续运行 012 脚本：
```cmd
python LOF012_generate_lof_data.py
```
**核心验证点**：
打开你的测试基金表格 `data\LOF_160719_history.csv`（或其他 LOF 表格），重点检查：
1. `012` 是否正确地触发了 `013` 去 Woody 网页爬取了 `GLD` 和 `^GLD-EU` 的数据，并填回了 basic 表。
2. `012` 是否去东财和新浪补齐了 T-1 日的“LOF净值”和“LOF收盘价”。
3. **最核心的一步**：检查表格右侧的 **ETF静态估值** 列，看程序算出来的 T-1 日估值，是否和你刚才在 Excel 里手工计算的结果 **100% 吻合**！

### 第三步：手动启动后台服务 (可选，仅为实时数据准备)
如果你想把整套系统跑起来看网页，你需要开启后台服务。因为后台服务会持续运行并阻塞窗口，建议你**新开两个 CMD 窗口**（同样先 `cd /d D:\Study\codexTest\CodexLOFarb`），分别运行：

**在窗口 A 运行（启动维护面板服务 5002端口）**：
```cmd
python LOF01_admin_launcher.py
```

**在窗口 B 运行（启动行情引擎 5000端口）**：
```cmd
python LOF02_fetch_trade_data.py
```

### 第四步：手动生成监控 HTML 页面
回到你**最初的那个 CMD 窗口**，运行：
```cmd
python LOF03_generate_monitor_html.py
```

**最终验证**：运行完毕后，你可以直接双击目录下的 `lof_monitor.html`，或者在浏览器打开 `http://localhost:5000/`，看看在刚才完美的数据基石上，页面渲染出来的静态估值和 T-1 日溢价是否一切正常。
