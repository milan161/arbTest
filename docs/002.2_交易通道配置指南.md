# 🏦 交易通道配置与对接指南

系统目前支持三种 A股/LOF 交易通道，和一种美股交易通道。在前端网页的“交易沙盘”中，您可通过下拉框随意切换实盘通道。

## 1. 🟡 国金证券 MiniQMT (xtquant 原生直连)
- **原理**：直接调用 QMT 官方提供的 Python 极速底层库 `xtquant`。无需在 QMT 客户端中运行任何策略，系统直接从外部向 C++ 核心发单。
- **配置要求**：
  - **安装路径**：默认配置为 `D:\GJQMT`（需包含 `bin.x64` 和 `userdata_mini` 目录）。
  - **资金账号**：默认配置为 `66655836`。
  - **运行前提**：启动程序前，**必须先登录并保持国金 MiniQMT 客户端开启**。
- **优势**：极速、稳定、支持同步获取订单编号（seq）和详尽的资金回调。

## 2. 🔵 银河证券 QMT (Socket 8888 桥接)
- **原理**：部分券商（如银河）客户端对外部 `xtquant` 库进行了封锁，只能在 QMT 客户端内部编辑器中运行 Python 代码。我们采用在客户端内部建立一个 TCP Socket 服务器（端口 8888）来接收外部指令。
- **配置要求**：
  - **客户端设置**：需要在银河 QMT 策略编辑器中加载并运行 `readers/qmt_socket_server.py`。
  - **通信协议**：主程序通过 `127.0.0.1:8888` 发送诸如 `BUY,162411.SZ,100,1.520` 的文本指令。
- **优势**：能突破券商的安全封锁，泛用性极强。

## 3. 🔴 通达信 (极速内存注入)
- **原理**：依托于通达信的 `tqcenter` 插件 API，在同进程内存中直接操作通达信客户端的挂单窗口。
- **配置要求**：
  - **安装路径**：`D:\new_tdx64\PYPlugins\user`。
  - **运行前提**：通达信客户端需登录并常驻后台。
- **优势**：适合作为 QMT 的后备容灾方案。并且兼具拉取 A 股实时快照的行情兜底功能。

## 4. 🌍 盈透证券 IB (Gateway / TWS API)
- **原理**：连接本地盈透交易网关，用于处理美股 ETF 现货（如 GLD, USO, SPY）和相关 CME 期货的对冲端下单。
- **配置要求**：
  - **端口**：系统会自动尝试遍历 `4001`, `4002`, `7496`, `7497`，无缝兼容 Gateway 与 TWS。
  - **设置**：客户端需要在“API设置”中勾选“启用 ActiveX 和 Socket 客户端”。
  - **盘前盘后**：系统发单时已默认附带 `outsideRth = True`，支持触发夏令时 08:00 开始的 OVERNIGHT 夜盘时段成交。

---

## 🛠️ 修改账号与路径
若您在未来变更了券商或安装位置，请直接修改 `readers/trade_manager.py` 中的 `_init_guojin_qmt()` 或 `_init_tdx()` 对应的路径和账号硬编码，重启主程序即可生效。
