# 🚀 LOF基金套利监控系统 (LOF Fund Arbitrage Monitoring System)

## 📑 项目概述
本项目是一个面向A股、美股（海外LOF底层资产）及相关期货市场的高度自动化套利监控与交易执行系统。系统集成了多源实时行情（东财SSE、新浪、盈透IB Gateway），并支持多通道的一键打单及实时估值推演。

## 🎯 核心功能
1. **全天候实时行情**：支持A股盘中、美股盘前及夜盘长连接（IB OVERNIGHT）、CME期货及国内沪银连续报价。
2. **多通道一键打单**：首创双轨制 QMT 架构（支持银河 Socket 桥接与国金原生 xtquant 直连），外加通达信极速内存打单与 IB 外盘打单。
3. **动态毫秒级估值**：首创前台纯 JS 无刷新估值推演，根据底层 ETF 与期货的盘口和实时汇率，精准测算当前 LOF 溢价率。
4. **纯数据驱动架构**：历史数据更新采用“缺了就补、有了不碰”的稳健策略，自动锚定 T-1 交易日，无惧节假日与时差错配。
5. **模块化与高可用**：交易引擎、数据获取、前端渲染等核心模块已完全解耦，并内置了端口防碰撞、通道自动降级等健壮性机制。

## 🏗️ 核心业务脚本流转
- `LOF00_input_LOF_info.py`：可视化配置中心（端口 5001），更新 `lof_config.yaml`。
- `LOF01_admin_launcher.py`：后台任务调度面板（端口 5002），调度数据抓取。
- `LOF011_generate_basic_data.py`：基础数据聚合器，整合 API 与网页爬虫。
- `LOF012_generate_lof_data.py`：LOF历史净值清洗与对账。
- `LOF02_fetch_trade_data.py`：**系统主心脏（端口 5000）**，承载 WebSocket 服务、路由分发与底层 API 守护。
- `LOF03_generate_monitor_html.py`：前端监控沙盘生成器，采用门面模式委派子 Manager 渲染界面。

## ⚙️ 核心算法
### 1. 对冲数量 (Hedge Quantity) 的双轨推演算法
在量化沙盘中，当系统预设固定的人民币本金时，必须**极其精准地推算出敞口（Delta）闭合状态下，中美两市所需的委托股数。系统针对不同类型的资产设计了两套独立的逆向物理推导模型，其核心锚点全部建立在 API 解析而来的物理 `对冲值 (Hedge)` 之上。**

#### 1.1 纯ETF与指数类（如 162411, 161127, 161125, 161130）
**算法逻辑：美股定 A 股**。由于此类资产成分高度单一，等效追踪效应极强，须保证美股股数的整数下探精准。
1. **计算临时拟合股数**：`临时数量 = 投入金额 / LOF盘中实时交易价`
2. **锁定美股实际委托头寸**：`美股对冲数量 = ROUND(临时数量 / 对冲值)` （四舍五入，保底 1 股）
3. **反推锁定A股委托头寸**：`LOF实际交易数量 = ROUND((美股对冲数量 × 对冲值) / 100) × 100` （A股受限100股倍数规则，依此凑整）

#### 1.2 商品类（如 160719 黄金, 160723 原油）
**算法逻辑：A股定美股**。大宗商品 LOF 底层常由全球一篮子资产拼接转换，A 股为实际执行主端，需先强制凑齐境内手数，后用海外衍生品兜底。
1. **锁定A股实际委托头寸**：`LOF实际交易数量 = ROUND((投入金额 / LOF盘中实时交易价) / 100) × 100`
2. **被动派生美股委托头寸**：`美股对冲数量 = ROUND(LOF实际交易数量 / 对冲值)` （四舍五入，保底 1 股）
*(注：大宗商品本身缺乏直接对冲比参数，其底层物理“对冲值”将在 012 构建中，通过 `Woody动态校准值 ÷ 基金实际合约仓位比例` 重建并自动固化。)*
