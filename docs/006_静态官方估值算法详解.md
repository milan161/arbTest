# 静态官方估值算法详解（含基准日选取）

## 0. 什么是静态官方估值？

静态官方估值是**用昨天（T-1日）的官方净值为锚点**，结合今天的价格变动估算出来的估值。它的目的是：等今天晚上基金公司公布净值后，进行对比验证，如果误差小，说明我们的估值算法正确。

---

## 1. 核心：基准日的选取

基准日（Anchor Date）是计算静态估值的关键！选取原则：

### 1.1 基准日定义
从基金历史数据中，找**最近一个有官方净值的有效交易日**作为基准日（记为 T-1）。

### 1.2 选取逻辑（代码实现）
```python
# 按日期降序排序（最新的在前）
lof_df_sorted = lof_df.sort_values('date', ascending=False).reset_index(drop=True)

# 遍历找第一个有净值的日期
base_date = None
base_nav = 0.0
base_exchange_rate = 7.1

for _, row in lof_df_sorted.iterrows():
    nav_val = row.get('nav', 0)
    if isinstance(nav_val, (int, float)) and nav_val > 0:
        base_date = row['date']
        base_nav = nav_val
        base_exchange_rate = row.get('exchange_rate', 7.1)
        break
```

## 2. 两种静态估值的区别

我们系统有两种静态估值，**算法公式完全一样**，唯一区别是**价格数据源不同**：

| 估值类型 | 价格数据源 | 适用场景 |
|---------|-----------|---------|
| ETF静态估值 | 真实ETF价格（如GLD、USO、XOP等） | 有真实ETF数据时 |
| 期货静态估值 | 期货收盘价（如GC、CL等） | 无ETF数据或想用期货验证时 |

---

## 3. 核心计算公式

### 3.1 公式拆解

```
T日估值 = T-1日净值 × [1 + 仓位 × (价格变动 × 汇率变动 - 1)]
```

其中：
- **价格变动** = T日价格 / T-1日价格
- **汇率变动** = T日汇率 / T-1日汇率
- **仓位** = 基金的权益仓位比例（0~1）

### 3.2 完整示例（黄金LOF）

假设：
- T-1日净值 = 1.2000
- 仓位 = 95% = 0.95
- T-1日GLD价格 = 200.00
- T日GLD价格 = 202.00
- T-1日汇率 = 7.2000
- T日汇率 = 7.1800

计算：
```
价格变动 = 202.00 / 200.00 = 1.01
汇率变动 = 7.1800 / 7.2000 = 0.9972
净值变动 = 0.95 × (1.01 × 0.9972 - 1) ≈ 0.95 × (1.0072 - 1) ≈ 0.95 × 0.0072 ≈ 0.00684
T日估值 = 1.2000 × (1 + 0.00684) ≈ 1.2082
```

## 4. 多持仓ETF的处理

如果基金持仓有多个ETF（如黄金LOF同时持有GLD、SLV等），需要**按权重加权计算**：

### 4.1 权重加权公式
```
加权价格变动 = Σ(单个ETF价格变动 × 该ETF权重 / 总有效权重)
```

### 4.2 代码实现
```python
weighted_change_rate = 0.0
total_valid_weight = 0.0

for item in holdings:
    symbol = item['symbol']
    weight = item['weight']
    
    if weight <= 0 or weight < 2.0:  # 过滤权重过小的
        continue
    
    total_valid_weight += weight
    
    # 计算单个ETF的价格变动
    etf_change = current_price / base_price
    weighted_change_rate += etf_change * (weight / total_valid_weight)
```

---

## 5. 为什么需要两种静态估值？

| 原因 | 说明 |
|-----|-----|
| 交叉验证 | 用两种不同数据源计算，结果一致说明算法正确 |
| 数据兜底 | 当ETF数据缺失时，可以用期货数据替代 |
| 算法验证 | 为接下来的"实时估值"打基础（实时估值也是同样的公式） |

---

## 6. 验证时机

每天晚上基金公司公布净值后，对比：
- 基金公司公布的净值
- 我们计算的静态官方估值

如果误差在可接受范围内（比如 < 0.5%），说明我们的算法没问题！
