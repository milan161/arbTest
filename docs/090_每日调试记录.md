### 修复2：周末静态估值缺失（核心问题）

**问题**：黄金原油 TAB 中 160719（嘉实黄金）07-02 的 `static_val` 为 None。

**现象**：
- 07-02 的 nav=1.852, price=1.813, premium=-0.494%，但 static_val 为空
- 07-01 的 static_val=1.8234 正常
- 同属黄金原油的 162411（华宝油气）07-02 static_val=0.824 正常

**根因分析（两层）**：

#### 根因2a：周末模式跳过静态估值计算

`daily_updater.py run()` 方法中，周末分支（weekday 5,6）**只执行 `step4_fetch_lof_market()` 净值更新**，跳过了 `_step10_calculate_static_valuation()`。

- 工作日完整流水线：step4 净值更新 → step10 静态估值计算 → 全部写入 DB
- 周末流水线：step4 净值更新 → **直接结束**，step10 未执行

结果：周末更新了 nav/price，但 static_val 没有重新计算。07-02 是周四，理论上静态估值应该在前一个工作日（07-03 之前）就计算好了。但 07-02 的 nav 是在 07-03（周五）更新的，而 07-03 之后到周末都没有再跑 step10。

#### 根因2b：区域锚点权重列名不匹配（bfill 失效）

`static_valuation.py _calculate_history()` 第98-102行：

```python
# 权重列 bfill
for item in portfolio:
    sym = item.get('symbol', '').replace('^', '')  # "GLD-EU"
    w_col = f"{sym}_weight"  # "GLD-EU_weight" ← 没有 ^
    if w_col in df.columns:  # 实际列名是 "^GLD-EU_weight" → 找不到！
        df[w_col] = df[w_col].bfill()
```

**问题**：
1. portfolio 中的 symbol 是 `GLD-EU`（来自 lof_config.yaml）
2. merge ETF 价格时，第55行加了 `^` 前缀 → 列名变为 `^GLD-EU`
3. merge 权重时，列名变为 `^GLD-EU_weight`
4. 但 bfill 时第99行先去掉了 `^` → 找 `GLD-EU_weight` → **列名不匹配** → bfill 静默跳过

对于 07-02，basket weights 表中没有记录（07-01 之后没更新），权重列值为 NaN。bfill 失效后，`_deduce_valuation` 中 weight 为 NaN，导致矩阵公式计算结果为 None。

**验证**：
- 162411（华宝油气）有 basket 数据 → 权重列有值 → 不受影响
- 160719（嘉实黄金）07-02 无 basket 权重 → bfill 失败 → static_val 为 None

**修复**：
1. **daily_updater.py**：周末模式也执行 `_step10_calculate_static_valuation()`
2. **static_valuation.py**：权重 bfill 时处理区域后缀（加 `^`），与 merge 时保持一致

```python
# 修复后
for item in portfolio:
    sym = item.get('symbol', '').replace('^', '')
    # 处理区域后缀 — 和 merge 时保持一致
    if any(suffix in sym for suffix in ['-JP', '-EU', '-HK']):
        sym = f"^{sym}"
    w_col = f"{sym}_weight"
    if w_col in df.columns:
        df[w_col] = df[w_col].bfill().fillna(item.get('weight', 0))
```

**验证结果**：修复后 10 只黄金原油基金 07-02 static_val 全部正确：
- 501018: 1.4547, 160723: 1.6725, 161129: 1.3837
- 160216: 0.6395, 161815: 0.984, 160719: 1.8499
- 161116: 1.5385, 164701: 1.613, 165513: 0.9767, 163208: 1.1927

07-03 为 None 是正确的（美国独立日假期，无 ETF 价格）。

### 修复3：fund_service.py 估值误差改为绝对差值 + 增加 hedge 列

**问题**：用户反馈前端"估值误差"列显示百分比不直观，且单一 ETF 基金（魔法公式）需要显示 hedge 值。

**修复**：
1. `fund_service.py get_fund_history()`：
   - SQL JOIN 增加 `f.hedge` 字段
   - `val_error_pct` 从百分比 `(static_val/nav - 1)*100` 改为绝对差值 `static_val - nav`
2. `Dashboard.vue` 历史记录：
   - 判断是否有 hedge 数据，有则显示"对冲值"列
   - `knownKeys` 增加 `'hedge'`
   - 估值误差显示改为 `v.toFixed(4)`（非百分比格式）

**注意**：此修复仅影响有 hedge 数据的基金（主要是 QDII 欧美跟踪单一 ETF 的基金，如 161125、161126 等）。黄金原油基金使用矩阵公式，不需要 hedge 列。
