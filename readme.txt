一、 简介
本程序是一个用来帮助新手学习如何在AI辅助下进行Python编程的测试软件，软件中所有数据均有可能是错误的，非实时的，不应被误导而作为实际投资所用！！！请注意风险！！

本程序选用跨市场基金折溢价套利监控项目作为开发测试，采用"大一统底层基座 (arbcore) + SQLite WAL并发数据库 + ArbDashboard 核心看板"架构，目标是尽力争取实时监控黄金原油、QDII 欧美、QDII 亚洲、国内 LOF、白银等各类上市基金的实时溢价与估值偏差。

本程序的估值算法核心基于 palmmicro 网站及公众号的技术体系，并使用了 Woody 提供的 API 接口获取关键估值因子（仓位、权重、校准值等）。为了完整使用本程序，欢迎联系 Woody 获取 API 接口（详见下方说明）。

二、 环境要求

在开始之前，请确保已安装以下软件：

| 软件 | 推荐版本 | 下载地址 |
|------|---------|---------|
| Python | 3.11 或 3.12 | https://www.python.org/downloads/ |
| Node.js | 18+（推荐 20 LTS） | https://nodejs.org/ |
| npm | 随 Node.js 安装 | — |

验证安装：
  >> python --version   # 应显示 Python 3.11.x 或 3.12.x
  >> node --version     # 应显示 v18.x.x 或更高
  >> npm --version      # 应显示 9.x.x 或更高

三、 安装与启动

【安装流程图】
   安装 Python 3.11/3.12 → 安装 Node.js 18+
        ↓                       ↓
   pip install -r backend/requirements.txt
        ↓
   cd frontend && npm install
        ↓
   启动后端 → 启动前端 → 打开浏览器
   (python main.py)  (npm run dev)

【首次使用前必读】
  从 GitHub 下载的 `database/arb_master_share.db` 需要重命名为 `arb_master.db`：
    >> cd <项目目录>\database
    >> ren arb_master_share.db arb_master.db

【一键启动（推荐）】
  在主目录下，双击运行以下批处理文件：
  >> start_dashboard.bat
  启动脚本会自动检查 Python/Node.js/npm 环境、自动 pip install 和 npm install，
  然后启动后端（8000端口）+ 前端（5173端口），并自动打开浏览器。

【手动启动】
  1) 安装 Python 依赖：
     >> cd backend
     >> pip install -r requirements.txt

  2) 安装前端依赖：
     >> cd frontend
     >> npm install

  3) 启动后端（端口 8000）：
     >> cd backend
     >> python main.py

  4) 启动前端（端口 5173）：
     >> cd frontend
     >> npm run dev

* 注：浏览器打开 http://localhost:5173/ 即可访问看板。
* 后端 API 文档：http://localhost:8001/docs

四、 项目结构

三层架构：
   展示层 (Vue 3 + Vite)           ← HTTP/REST API

   公用库层 (arbcore)              ← Fetchers / Calculators
   数据层 (SQLite + 通达信/QMT/新浪)

基金分类：
   黄金原油 (162411→XOP, 162415→GLD)
   QDII 欧美 (161126→SPY, 161127→QQQ)
   QDII 亚洲 (港股/日股基金)
   国内 LOF (501018 等)
   白银 (161116)

五、 核心功能

1. 套利看板（Dashboard）
   - 全场监控：黄金原油、QDII 欧美/亚洲、国内 LOF、白银
   - 数据列：现价、涨跌幅、实时估值、实时溢价率、静态估值、静态溢价率
   - 刷新频率：前端每 15 秒轮询一次

2. 深度分析（Analysis）
   - 实时曲线：基金现价 vs 实时估值
   - 溢价率柱状图：红绿柱状图（0 轴为中心）
   - 测算沙盘：手动输入假设价格，实时推演溢价率

3. 自动交易（AutoTrade）
   - 规则引擎：网格化规则，自动触发买卖
   - 信号监控：当前溢价率距离触发阈值的进度条
   - 守护线程：每 5 秒扫描全场

4. 实盘对账（Ledger）
   - 交易记录：自动记录交易明细
   - 赎回提醒：T+3 工作日快速赎回（自动跳过周末）

5. 数据管理（Data）
   - 手动触发数据同步
   - 健康监控：数据库体积、采样点数

6. 系统配置（Settings）
   - 数据源优先级：通达信、QMT、新浪、东财
   - 拖拽调整、启用/禁用、测试连接

六、 关于 Woody API 与数据源的开源说明

本程序以开源形式发布于 GitHub 公共仓库。程序内部包含一套完善的数据获取流水线
(daily_updater)，按照"VPS 云端优先 -> 直调 API -> 网页爬虫兜底"的降级策略运行。

运行完整的数据流水线（含 Woody 因子、官方汇率、期货结算价等）依赖以下两项配置：

  1. Woody API Key（需要向 Woody 申请获取）
  2. 个人账户密钥文件 `arbcore/config/account_private.py`（不上传 Git）

如果您是首次下载本程序，未配置上述密钥，程序将自动触发以下降级行为：

  - VPS 云端数据通道：跳过（无密钥无法连接）
  - Woody API 直调：跳过（无 API Key）
  - 网页爬虫兜底：自动尝试，但因 Woody 网站需登录验证，大概率超时失败

上述降级是程序的安全设计，确保您不会在缺少必要凭据时拿到错误或过时的估值数据。
但与此同时，如果您只是想体验前端看板的实时行情展示与沙盘推演功能，**无需任何密钥**，
直接启动后端即可查看主看板的基本功能（A 股/美股实时行情、折溢价监控、手动沙盘分析）。

若希望运行完整流水线以获得正确的历史估值数据，建议直接联系 Woody，申请个人 API Key（推荐）
 访问 palmmicro 网站或公众号，获取 Woody API 的使用方式和信息， 配置到 `arbcore/config/account_private.py` 后，程序即可正常运行。

七、 极速配置个人私密密钥 (关于 Woody API 与交易账号)
如果需要运行盘后更新流水线以同步最新历史数据，需要配置个人密钥。
请在本地创建或编辑以下文件：
  >> 配置文件路径：`arbcore/config/account_private.py`

可以在该文件中配置以下保密参数（此文件已配置在 Git 屏蔽名单中，绝对不会被上传）：
  - `BOT_Key` = "您的Woody_BOT_Key"   (用于运行盘后历史数据流水线)
  - `WOODY_USERNAME` = "您的邮箱" / `WOODY_PASSWORD` = "您的密码" (网页爬虫备用登录)
  - 银河/国金 QMT 相关账号和密码。

* 注：普通用户即便不填写任何密钥，直接启动主看板 (ArbDashboard) 也是可以完全正常查看实时行情、折溢价与进行沙盘推演的。

八、 手动调试与重启

如果启动后遇到问题（如前端白屏、数据不更新等），可以手动重启前后端：

  1) 重启前端（前端页面不显示或样式异常时）：
     >> cd D:\Study\arbTest\ArbDashboard\frontend
     >> npm run dev          # 开发模式（热重载）
     >> npm run build        # 构建生产版本（启动脚本会自动调用此命令）

  2) 重启后端（数据不更新或报错时）：
     >> cd D:\Study\arbTest\ArbDashboard\backend
     >> python main.py       # 启动后端服务（端口 8000）

  3) 一键重启（推荐）：
     >> 双击 D:\Study\arbTest\ArbDashboard\start_dashboard.bat
     该脚本会自动执行 npm run build + python main.py + npm run dev

* 注：修改前端代码后需要执行 npm run build 才能生效；修改后端 Python 代码后需要重启 python main.py 才能生效。

九、 各数据源关系与功能说明 (四级瀑布流设计)

程序在获取行情时会自动采用降级和备选机制，不需要同时开启所有交易软件：

1. 国内 A 股及期货行情 (四级瀑布流)：
   * 银河 QMT (最推荐，Socket 直连，速度最快)
   * 通达信 TQ 接口 (内存直连，需要启动通达信客户端并配置 TQ 插件)
   * 国金 QMT (xtquant 极速策略库)
   * 新浪 API (低频轮询，无需运行任何客户端，自动作为最终兜底)

2. 美股/港股标的实时现价 (双通道)：
   * 盈透证券 (IB Gateway / TWS)：美股“重点关注的黄金原油、QDII欧美” ETF 夜盘/实时主通道。
   * 富途 OpenD (FUTU)：备用通道，若无 IB 账户，开启富途牛牛 API 后自动接管行情。

3. 历史净值与估值数据 (云端备份 + 本地爬虫)：
   * Woody API：估值计算的核心锚点与历史定价基准。
   * 腾讯/新浪接口：国内 LOF 与美股历史 K 线/成交量的自动化获取。

4. 重新连接机制说明（V10.0 重要变化）

  *   程序启动时，通达信、国金 QMT、银河 QMT、IB、富途等所有需要客户端的数
    据源都**不会自动连接**（避免启动阻塞）。只有新浪、腾讯等纯 API 源会自动
    连接作为兜底。

  *   如需启用某个数据源，请在主看板（Dashboard）页面**顶部**找到对应的连接
    标签（如"通达信"、"IB"、"银河QMT"、"富途"等），**点击该标签**即可触发
    连接。连接成功后标签会从红色变为绿色，无需重启后端程序。

  *   标签颜色含义：
      红色 = 未连接（点击可重连）
      绿色 = 已连接（正常工作中）
      黄色 = 正在连接中

  *   如果点击后仍然无法连接，请确认：
      1. 对应客户端（如通达信、QMT、IB Gateway、富途 OpenD）已经启动
      2. 客户端已登录并处于正常状态
      3. 如果是周末非交易时段，QMT 可能处于休眠状态，部分功能受限
